Tip #0: Update IE

Keep IE up to date with Windows Update

  • Typically contain security fixes and high impacting bugs
  • For the first time the April 8th update will include features
    • Also known as Windows 8.1 Update 1 but applies to IE11 on Win7 as well
    • Lots of new F12 features
  • In this talk if you see "new" in the corner then the feature was added in the April 8th update

Tip #1: Just My Code

Never debug 3rd party code

  • Step only in your own code
  • Break in caught exceptions in your code only
  • Break next in your code
  • Mark your files:
    • File picker, tab header, MyCode.json

Tip #2: Source Maps

Using a minifier or language like TypeScript? Build source maps to make debugging easier

  • F12 now supports the Source Maps v3 spec!
    • Syntax highlighting for TypeScript, CoffeeScript and Script #
    • Dynamically enable and disable

Tip #3: TracePoints

Instrument code without having to redeploy it

  • Work like breakpoints but log to the console
  • Log strings, objects and inspect in the console
  • Access to locals

Tip #4: Never miss a console message

Turn on "Always record console messages"

Internet Options -> Advanced -> Browsing -> Always Record Console Messages (check)
(broken site)

Tip #5: Keyboard shortcuts

Be more productive by using the keyboard

  • When broken in the debugger use ctrl + shift + F5 to abort and refresh the page
  • Move between tools with ctrl + [ or ctrl + ]
  • Don't forget ctrl + # to jump to a tool (e.g. DOM Explorer is ctrl + 1)

Tip #6: Changebars

Keep track of changes for F12

  • Keep track of all the CSS changes you make through the tools
  • New "Changes" tab show all the changes
  • Revert the item back to it's original state

Tip #7: Heads Up Display

Monitor performance in real time




(ctrl + shift + u)

(slow site)

Tip #8: Instrument Your Code

Use the console.* API to


  • console.time/timeEnd - to quickly time scenarios
  • performance.mark - to delineate scenarios in the profiler
  • console.takeHeapSnapshot - to take a snapshot

Tip #9: Enterprise Mode

Update IE and keep your internal sites running


  • Runs sites in an IE8 emulation mode
    • Document mode at IE8
    • ActiveX enabled
    • Old JavaScript engine
    • A number of security improvements are disabled
  • The sites that run in this mode are configurable via XML and group policy
    • Use the Enterprise Mode Site List Manager tool to configure XML
    • More on TechNet

Summary


  • Lots of tips & tricks covered today
  • Top tip is to please update IE11 :)
