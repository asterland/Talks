Tip #0: IE 업데이트하기

윈도우 업데이트를 통해 IE를 최신버전으로 유지하세요.

  • 일반적으로 보안 픽스와 큰 영향을 미치는 버그 수정이 포함됩니다.
  • 4월 8일 업데이트에서는 처음으로 아래와 같은 기능들이 포함이 됩니다.
    • 윈도우 8.1 업데이트 1이라고도 알려져 있지만, 윈7의 IE11에도 적용됩니다.
    • 많은 F12 신기능이 추가 됩니다.
  • 슬라이드 화면의 우측 상단에 "new"라고 표시가 있으면 4월 8일 업데이트에 새롭게 추가된 기능입니다.

Tip #1: 내 코드만 디버깅하기

절대 서드파티 코드를 디버깅하지 마세요.

  • 내가 작성한 코드에서만 Step-in 합니다.
  • 내 코드에서만 예외를 잡아 중단합니다.
  • 내 다음 코드에서 중단합니다.
  • 내 파일을 지정할 수 있습니다:
    • 파일 피커, 탭 헤더, MyCode.json

Tip #2: 소스맵

Minifier나 TypeScript 같은 언어를 사용하세요? 그럼 소스맵을 만들어서 디버깅을 더 쉽게 하세요.

  • F12는 이제 소스맵 V3 스펙을 지원합니다!
    • TyppScript, CoffeeScript, Script #등을 위한 신택스 하일라이팅
    • 동적으로 허용 및 중단

Tip #3: 트레이스 포인트

다시 배포할 필요 없는 측정용 코드

  • 중단점처럼 동작하지만, 콘솔에 로그를 남깁니다.
  • 문자열이나 오브젝트를 로깅하고, 콘솔에서 조사할 수 있습니다.
  • 지역 변수에 접근 가능합니다.

Tip #4: 콘솔 메시지 안 놓치기

"개발자 콘솔 메시지를 항상 저장하기" 기능을 켜세요.

인터넷 옵션 -> 고급 -> 브라우징 -> 개발자 콘솔 메지시...(체크)
(고장난 사이트)

Tip #5: 키보드 단축키

키보드를 사용하여 생산성을 더 높이세요.

  • 디버거에서 중단됐을 때 Ctrl + Shift + F5를 눌러서 중지시키고 페이지를 새로고침 할 수 있습니다.
  • 툴 간에 ctrl + [ 또는 ctrl + ]로 이동할 수 있습니다.
  • ctrl + #를 눌러서 특정 페이지로 이동할 수도 있다는 점을 잊지 마세요.(예, DOM Explorer는 ctrl + 1)

Tip #6: 체인지 바(Changebars)

F12의 변경사항을 계속 추척하세요.

  • 툴을 통한 CSS 변경을 모두 추척할 수 있습니다.
  • 새로운 "Changes" 탭은 모든 변경사항을 보여줍니다.
  • 항목을 원래 상태로 돌릴 수 있습니다.

Tip #8: 코드를 측정하기

console.* API 를 사용하세요.


  • console.time/timeEnd - 시나리오의 시간을 빠르게 측정
  • performance.mark - 프로파일러에서 시나리오를 기술
  • console.takeHeapSnapshot - 스냅샷을 생성함

Tip #9: 엔터프라이즈 모드

IE를 업데이트하고 내부 사이트가 제대로 동작하게 하기


  • IE8 에뮬레이션 모드에서 사이트를 실행합니다.
    • IE8의 문서 모드
    • ActiveX 사용 가능
    • 구 버전의 JavaScript 엔진
    • 일부 보안 향상기능 꺼짐
  • 이 모드에서 실행하는 사이트는 XML과 그룹 정책으로 설정 가능합니다.
    • XML을 설정하기 위해서 Enterprise Mode Site List Manager 툴을 사용하세요.
    • 자세한 내용 TechNet
